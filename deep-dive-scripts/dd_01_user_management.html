1
Why User Management Is Your Starting Point

User Management is the foundation every other module sits on. The users you create here are the people who will own tickets, log time, get dispatched to jobs, process billing, generate reports, and administer the platform. The permission sets you assign determine what each of those people can see and do across every module. The groups you create control how work gets routed, who gets notified, and how the dispatch board is organized.

There is no meaningful work to do in Rev.io PSA until users exist. Everything downstream — System Settings ticket defaults, the Dispatch Board, ticket assignment, billing approvals, report access — references what you build here.

→ Connects ToUser groups are referenced directly in System Settings for default ticket routing. Permission sets gate access to every module: Billing, Quoting, Admin, Inventory, Customers, Reports, and Service. Users appear on tickets (assigned to), time logs (logged by), calendar events (assigned to), the dispatch board (scheduled), and all reporting dimensions.
▲ Platform ImpactPoorly configured permissions create two types of problems: users blocked from doing their job because access is too restrictive, or users with access to sensitive financial data, billing records, or admin settings they shouldn't see. Both are disruptive — one creates operational bottlenecks, the other creates security and compliance risk. Getting permissions right at setup is far easier than cleaning them up after the fact.

3
Permission Sets — Full Coverage

Permission Sets are the security backbone of Rev.io PSA. Every permission set is a collection of toggles that control access at a granular level — not just which modules a user can see, but what actions they can take within each module, what data they can view, and what they can modify or delete.

Navigate to Admin > User Management > Permission Sets. Click Add Permission Set. Name it by role type.

Permission Categories

  • Service Module: Create tickets, edit tickets, close tickets, delete tickets, view all tickets vs. only assigned tickets, manage ticket types and statuses
  • Customer Module: Create customers, edit customers, view customer billing data, access customer portal settings, merge duplicate accounts
  • Billing Module: View invoices, create invoices, post charges, void invoices, apply payments, view customer financial data, run billing reports
  • Inventory Module: View catalog, edit catalog, create purchase orders, receive inventory, manage warehouses
  • Quoting Module: Create quotes, send proposals, view accepted quotes, convert quotes to tickets or invoices
  • Admin Module: Manage system settings, manage users, manage permission sets, configure integrations, view audit logs
  • Scheduling: Access dispatch board, create calendar events, view all users' schedules vs. only own
  • Reports & Analytics: View standard reports, build custom reports, export data, access financial reports
  • Projects: Create and manage projects, view all projects vs. only assigned projects

Recommended Role Profiles

RoleSuggested Access Level
TechnicianService module (own tickets), time logs, own calendar, view-only catalog, no billing, no admin
DispatcherService module (all tickets), full scheduling/dispatch board, view customers, no billing, no admin
Account ManagerService module, customers, quoting, view billing, no admin, no inventory management
Billing AdminFull billing module, view service and customers, no admin, no inventory
System AdminFull access to all modules including Admin settings
Manager / OwnerFull access or full access minus ability to modify admin settings
❓ PromptWhat roles do you have on your team and what should each one be able to do? Let's build a permission set for each role right now.
💡 Pro TipStart with the most restrictive permission set (Technician) and build up from there. It's easier to grant additional access later than to remove it once people are used to having it.
→ Connects ToThe full list of available permissions is documented at help.psarev.io/en/article/available-permissions. Reference it for edge cases like 'can this role approve timesheets?' or 'can this role see cost vs. sell price on catalog items?'
4
User Groups — Structure and Strategy

User groups are how Rev.io PSA organizes your team for ticket routing, scheduling, and reporting. A group is a named collection of users. When a ticket comes in, it can be assigned to a group instead of an individual — the whole group sees it and any member can act on it.

Navigate to Admin > User Management > User Groups. Click Add User Group.

Key Concepts

  • Multiple group membership: A user can belong to any number of groups. A senior technician might be in 'Level 1 Support,' 'Level 2 Escalation,' and 'After Hours On-Call' simultaneously.
  • Default ticket routing: In System Settings, you select a Default Ticket User Group — the group that newly created tickets are assigned to automatically. This is how unassigned inbound tickets land in the right queue.
  • Dispatch board filtering: The dispatch board can be filtered by group, making it easy for a dispatcher to see only the techs relevant to their area or territory.
  • Reporting dimension: Ticket volume, time logged, and other metrics can be sliced by user group in Analytics.

Common Group Structures

Structure TypeExample Groups
By tierLevel 1 Support, Level 2 Engineering, Level 3 Escalation
By territoryNorth Region, South Region, Downtown
By skillNetwork Team, Hardware Team, Software Team
By scheduleDay Shift, Night Shift, After Hours On-Call

❓ PromptHow is your team structured operationally? Do different groups handle different types of work, different territories, or different customer segments?
⚠ Watch OutDon't over-engineer groups at setup. Start with the groups that represent your actual daily routing decisions. You can always add more as the team grows and workflows become clearer.
5
Managing Users Over Time

User management isn't a one-time setup task — it requires ongoing maintenance as your team changes. Here's how to handle common scenarios:

Ongoing User Management Tasks

  • Adding a new hire: Create the user, assign permission set and group(s), set default labor rate and hourly cost. Send them the login link and confirm they set their password.
  • Employee departure: Deactivate the user account immediately. Their historical data — tickets, time logs, notes — is preserved. Reassign any open tickets they owned. Remove them from user groups.
  • Role change: Update the user's permission set and group assignments. Review their active ticket assignments if their territory or team is changing.
  • Password reset: From the user's profile, use the Resend Invitation or Reset Password option. The new link expires in 12 hours.
  • Audit user access: Periodically review who has access to the Admin module and Billing module. Remove access that's no longer needed.
  • Permission set updates: When you change a permission set, the change applies to all users on that set immediately. Coordinate updates to avoid disruption.
💡 Pro TipDo a quarterly user audit: review the active user list, confirm all active users are still employed, verify permission sets still match job roles, and remove any stale access.

Action Items

  • Create all team members as users
  • Build one permission set per distinct role type
  • Create all user groups that represent your routing and scheduling structure
  • Set default labor rates and hourly cost on all technician users
  • Assign calendar colors to field technicians for easy dispatch board identification
  • Communicate the password setup process and 12-hour expiry window to new users
  • Document your permission set definitions for future reference
