1
When Projects Replace Tickets

Most work in Rev.io PSA lives on individual tickets. But some engagements are too complex for a single ticket — they span weeks or months, involve multiple team members, have distinct phases with dependencies, and need to be tracked as a whole rather than as isolated work orders. That's when Projects become the right tool.

A Project is a container that organizes related tickets into a structured hierarchy. All time, parts, and billing from those tickets roll up to the project level, giving you a complete view of progress and cost for the entire engagement without hunting through individual tickets.

→ Connects ToProjects connect to: Tickets (organized into phases within the project), Customer records (the project belongs to a customer), Users (project manager, plus users assigned to individual tickets), Time Logs (roll up to project-level reporting), Billing (ticket charges flow to billing as normal), and Analytics (project-level profitability and status reports).
2
Project Structure — The Hierarchy
LevelDescriptionExample
ProjectThe overall engagement'Acme Corp — Network Infrastructure Upgrade 2025'
PhaseA logical stage of the project'Phase 1: Assessment', 'Phase 2: Installation', 'Phase 3: Testing & Handoff'
TicketIndividual work orders within a phase'Configure core switches', 'Deploy access points', 'Test throughput'
TaskGranular to-do items on tickets'Document current network topology', 'Verify firmware versions'

Progress is tracked at each level. When all tickets in a phase are closed, the phase is complete. When all phases are complete, the project is ready to close. Closing the project archives the record while keeping all history accessible.

Creating a Project — All Fields

FieldDescription
Project NameDescriptive name including customer and engagement type
CustomerThe customer this project is for — required

StatusPlanning, Active, On Hold, Completed

BudgetOptional planned budget — compare to actual cost in reporting

❓ PromptLet's create a project together. Think of a current or upcoming multi-phase engagement. What's the customer, what are the main phases, and what does the first phase involve?
4
Building Phases and Adding Tickets

Once the project is saved, click Add Phase. Name the phase to reflect the logical stage of work it represents. You can add as many phases as the project requires, and reorder them by drag-and-drop.

Within each phase, add tickets two ways:

  • Link existing tickets: If tickets for this work already exist, search for and add them to the phase.
  • Create new tickets: Create new tickets directly from the phase — they're automatically linked and inherit the customer from the project.

Tasks can also be added directly to phases for smaller action items that don't need a full ticket. These show in the phase's progress calculation.

💡 Pro TipCreate a project template for your most common project type. Once you've built one project with the right phases and standard tickets, use it as a reference when setting up similar projects. PSA doesn't have formal project templates, but a documented structure speeds up setup significantly.
5
Tracking Progress and Project Economics

The project overview shows: phases and their ticket counts, status of each ticket, overall percentage complete (based on closed vs. open tickets), total time logged, and total charges accumulated.

Budget vs. Actual

If you enter a Budget on the project, PSA tracks actual spend (time + parts) against that budget. This gives you an early warning when a project is trending over budget — before it becomes a financial problem.

Time Tracking on Projects

All time logged on tickets within the project rolls up to the project level. You can view total hours by phase, by technician, and by time type (labor, travel, etc.). Compare planned hours (from the quote that generated this project) to actual hours to understand efficiency.

💡 Pro TipReview project economics weekly during active engagements. If you're 80% done on time but 95% through your budget, you have a problem. Catching it early lets you have a proactive conversation with the customer rather than a surprise at billing.
6
Closing a Project

When all phases are complete and the work is done, close the project. Set the status to Completed and add a completion date. The project moves to your closed projects list, but all tickets, time logs, notes, and billing history remain accessible.

After closing, you can generate a project summary from the project record — a consolidated view of all work done, time spent, and charges billed. Useful for customer-facing project review conversations.

Action Items

  • Identify current or upcoming engagements that need project tracking
  • Create a project for each one with a clear phase structure
  • Add all relevant tickets to each phase
  • Set budgets on projects where cost control is important
  • Review project economics weekly during active projects
  • Build a standard phase template for your most common project type
